# fix(map-sync): stabilize terrain delivery and native runtime

## 修改目标

修复 Xaero 原生路径点管理页交互、世界地图地表颜色与空洞回填，并消除 Minecraft 1.17.1 在 Java 25 上运行旧 LWJGL 导致的客户端原生崩溃。

## 修改内容

- 将旧 RGB 颜色迁移为 Xaero 22.11.1 的 `0..15` 调色板索引；`0x55FFFF` 精确迁移为索引 11。
- 在服务端加载、创建和客户端落地三处防御非法颜色。
- 在 Xaero 原生 `GuiWaypoints` 页面加入可见性状态、公开共享、条件队伍共享和删除共享按钮。
- 操作区按当前可见按钮弹性均分；英文标签按内容宽度换行，不再保留隐藏 TEAM 按钮的空槽。
- 在 Xaero 列表点击处理前消费操作按钮事件，修复按钮高亮但回调不执行。
- 点击时读取 Xaero 当前唯一选中的原生路径点、显示世界、集合、坐标、符号和颜色。
- 创建者客户端以隐藏 UUID 绑定并复用原生路径点；其他客户端维护服务器同步副本。
- 已共享路径点使用独立绘制的锁图标，不再向名称写入可能乱码的字符；共享和 Xaero 修改按钮变灰，删除仍可确认执行。
- 客户端会把被本地编辑的远端共享点恢复为服务器版本，且不创建重复副本。
- 修正 Xaero World Map 1.25.1 的区域/区块加载状态反射签名和保存队列状态。
- 地图 tile 注入改用 Xaero 原生加载门控，立即失效旧缓存汇总状态，并在原生 60 秒纹理准备窗口后保存；保存、缓存或 writer 暂停中的区域无副作用重试。
- 已持久化瓦片不再被当前会话探索状态拒绝；不可用瓦片使用独立响应释放客户端请求槽位。
- 服务端启动时从 `.tile` 缓存重建缺失索引，并定时保存探索状态和地图索引。
- 修正地表高度的二次减一错误，直接采样高度图返回的顶部方块；旧采样 tile 通过独立版本标记进入持久化重建队列。
- 服务端启动时将 `explored - indexed` 的历史缺失区块幂等加入重建队列；每 tick 可扫描 64 个候选，但实际渲染数量仍受原预算限制。
- Merkle 子节点改为单次建树批量查询；分片组装、CRC 和解压保留在网络线程，主线程只处理最终 payload 与 Xaero 注入。
- 客户端为 Xaero 未准备好的区域维护有界延迟应用队列，不再因单一区域短暂不可用清空全局同步状态；旧 revision 不能覆盖新数据。
- Gradle/Loom 保持在 Java 21+ 运行，Minecraft 客户端和服务端 JavaExec 任务固定使用 Java 17。
- 移除 `]` 快捷键、独立管理页、手工新建页和本地导入页。
- TEAM 路径点加入 Xaero 客户端 reconcile。
- 更新回归测试、本地联调步骤、README 和预发布版本 `2.1.0-beta.7`。

## 不包含的内容

- 不升级 Minecraft、Fabric 或固定的 Xaero 版本。
- 不改变同步协议号或地图格式号。
- 不自动上传任何未选择的私人路径点。
- 不导入或改写 `xaeromap-origin/Data`；它仍只作为原始 Mod 数据参考。

## 风险等级

Medium

## 行为变化

修改前：按钮可能保持灰色或无法点击，隐藏 TEAM 后仍按四槽布局，锁字符显示方框；地表被采样成泥土色，未加载 dirty 项会阻塞当前区域，历史 explored 区块没有索引；B 客户端在 Java 25 的 GLFW/OpenAL 原生层崩溃。

修改后：原生列表选择会即时驱动操作状态；共享点显示锁图标且不可编辑，可经确认删除；草地与水体按正确顶部方块渲染，历史缺失索引持久回填，当前加载区块不会被未加载队列饿死；开发运行任务使用 Java 17。

## 评测结果

基线结果：两份客户端崩溃报告为 `COLORS[5636095]` 数组越界；beta.5 的 B 客户端另在强制保存未准备纹理缓存时触发 `Trying to save cache ... is not prepared`。

修改后结果：构建和 145 个测试通过；B 客户端使用 Java 17 完成 848 条冷同步修订，目标坐标 `26, -717` 在十几秒内由碎片地图热刷新为连续绿色地表；服务端启动识别并持久排队 727 个历史缺失索引，未出现新的 `hs_err`。

差异：高负载活动信号仍需大型破坏场景压力验收。

## 性能和成本

响应时间：共享点击只读取一个原生路径点并发送一条已有网络消息。地图任务默认每 tick 扫描最多 64 个候选、渲染 1 个；历史缺失任务不会主动加载区块。网络分片校验和解压不再占用渲染线程。

Token 消耗：不涉及模型调用。

工具调用次数：每次明确点击产生一次创建或删除请求。

## 安全影响

是否增加权限：否。

是否增加数据访问：只在点击时读取 Xaero 当前选中的一个路径点。

是否增加网络访问：不增加外部网络访问；新增一个 Fabric 服务端到客户端的瓦片不可用响应通道。

是否涉及敏感信息：不涉及密钥；路径点坐标仍按玩家明确操作提交。

## 发布方式

预发布版本，先在测试服务器灰度部署并观察 Xaero 保存线程、同步耗时与 MSPT。

## 回滚方式

停止服务端，回退本提交和 Mod JAR，并恢复对应的 `<world>/xaero-mapsync_r/` 数据备份。旧版本重新启用前必须确认服务端和客户端 Xaero 文件中不存在大于 15 的受管路径点颜色。
